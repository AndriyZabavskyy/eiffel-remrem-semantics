Eiffel REMReM Semantics

Eiffel REMReM Semantics is designed to implement new Eiffel messaging standard.

Eiffel REMReM Semantics is licensed under Apache License 2.0.

Usage

Please refer to eiffel syntax and eiffel vocabulary for usage.
Please note that meta.id, meta.time, meta.version are generated by REMReM Generate.

Messages Table

Below is a table of current official Eiffel2.0 events along with their templates and routing keys.
The routing key is concatenated from five entities: the event protocol, family, the event type, any custom tag and domain.
The routing key format for REMReM Semantics:
eiffel.<family>.<type>.<tag>.<domain>

  • <family> is a family name for a group of events, e.g. "activity".
  • <type> represents the event, e.g. "triggered".
  • <tag> is a field which carries no meaning for Eiffel itself, but may be used by users to implement a custom categorization scheme on which message routing may be applied. The tag must NOT contain any dots, as this will deny the receiver the ability to anticipate the word sequence of the binding key, and must not exceed 16 characters in length. Defaults to "notag".
  • <domain> The domain from which the message is sent. The value of the domain will fetch from eiffel event if empty the value is fetch from the REMReM Publish configuration. domain can be suffixed by a user domain part that the user can specify with -ud option. For more information on domain, see this link.

The routing key for EiffelActivityStartedEvent would be eiffel.activity.started.notag.eiffeltestdomain
